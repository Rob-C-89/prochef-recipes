# User Stories

User stories extracted from the [ProChef Recipes GitHub Project](https://github.com/users/Rob-C-89/projects/16).

---

## #1 - View a Recipe
**Status:** Closed | **Priority:** Must-have

As a Site User, I can click on a recipe post so that I can read the full text.

### Acceptance Criteria
- When a recipe title is clicked on a detailed view of the recipe is seen.

---

## #2 - View Comments
**Status:** Closed | **Priority:** Must-have

As a site user, I can view comments on each recipe post so that I can read the conversation.

### Acceptance Criteria
- User can click on the comment thread to read the conversation.

---

## #3 - Add Comments
**Status:** Closed | **Priority:** Must-have

As a logged-in user, I can add, delete and edit my own comments to contribute to the conversation around each recipe post.

### Acceptance Criteria
- Logged-in users can post their own comments on each recipe post.
- Logged-in users can edit their own comments.
- Logged-in users can delete their own comments.

---

## #4 - Like a Recipe Post
**Status:** Open | **Priority:** Won't-have

As a logged-in user, I can 'like' a recipe post to show the poster and other users that the post has been valuable to me.

### Acceptance Criteria
- Logged-in users can 'like' a recipe post.
- Number of likes are displayed on each recipe post.

---

## #5 - Register an Account
**Status:** Closed | **Priority:** Must-have

As a user, I can register an account on the website so that I can access all the features of the website.

### Acceptance Criteria
- Users can create an account with an email and password.
- User creates a username on registration.
- User can now log-in and access more features than a non-logged in user.

---

## #6 - Create a Profile
**Status:** Closed | **Priority:** Could-have

As a logged-in user, I can create a profile for the site, allowing me to showcase my bio and credentials to other users.

### Acceptance Criteria
- Logged in users can create a profile on registration.
- Profile can be viewed by other users, displaying personalised information and details.

---

## #7 - Search Recipes
**Status:** Open | **Priority:** Should-have

As a user, I can use a search function so that I can easily find the recipe I am looking for.

### Acceptance Criteria
- A search bar function returns a filtered list of recipes containing the searched for keywords.

---

## #8 - Create a Recipe
**Status:** Closed | **Priority:** Must-have

As a logged-in user, I can create and post my own recipes to contribute to the website content and community.

### Acceptance Criteria
- Logged-in users can create and post their own recipes.
- New recipes are displayed on the website.

---

## #9 - Admin Site Management
**Status:** Closed | **Priority:** Must-have

As a logged-in superuser/admin, I can delete recipe posts and users to maintain website quality.

### Acceptance Criteria
- Admin can delete any recipe post or comment.
- Admin can delete users.

---

## #10 - Delete Own Comments
**Status:** Closed | **Priority:** Must-have

As a logged-in user, I can delete my own comments to have control over my contributions to the site.

### Acceptance Criteria
- As the author of a comment, logged-in users can delete their own comments.

---

## #11 - Edit Comments
**Status:** Open | **Priority:** Could-have

As a logged-in user, I can edit my own comments to have control over my contributions to the site.

### Acceptance Criteria
- As the author of a comment, logged-in users can edit their own comments.

---

## #12 - Contact Website Owners
**Status:** Closed | **Priority:** No label assigned

As a logged-in website user, I want to be able to contact the site-owners for any number of reasons, from reporting issues, suggesting changes to the user experience, etc.

### Acceptance Criteria
- User can follow a link to a form, which will send a message to the site admin.
